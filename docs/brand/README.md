# Brand System

This directory is the release-facing brand source of truth for BLKBX Lab.

Core truth:

- short tagline: `Open-source Ink Receipt gates for accountable AI agents.`
- product line: `qwen35` is the installed deterministic demo. Receipt gates are the standard.
- technical strapline: `Signed actions · Verifiable receipts · Local policy gates`
- voice: precise, direct, earned, grounded

Canonical files:

- [`tokens.json`](tokens.json): color, typography, spacing, and naming rules
- [`tokens.css`](tokens.css): docs and README token implementation
- [`usage.md`](usage.md): short engineer guidance for release-facing copy and visuals
- [`release-copy.md`](release-copy.md): launch-ready copy blocks for README, release notes, and packaging-facing descriptions

Launch asset kit:

- [`../../assets/brand/og-card.svg`](../../assets/brand/og-card.svg): source-of-truth Open Graph card
- [`../../assets/brand/og-card.png`](../../assets/brand/og-card.png): GitHub social preview export for the live repo host
- [`../../assets/brand/launch-card.svg`](../../assets/brand/launch-card.svg): release announcement card
- [`../../assets/brand/release-header.svg`](../../assets/brand/release-header.svg): release note header
- [`../../assets/brand/badges/`](../../assets/brand/badges): proof and status badges
- [`../../assets/brand/icons/`](../../assets/brand/icons): launch icon subset

Naming rule:

- use `BLKBX Lab` for product copy
- use `mechlab-sdk` for install and published-package language
- use `blkbx-lab` for primary CLI examples
- use `blkbx_lab` for the primary Python namespace
- keep compatibility aliases and pre-BLKBX names inside migration-specific material only
