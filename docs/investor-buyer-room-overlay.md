# MAND8 Investor/Buyer Room Overlays

The room overlays are indexes over the shared evidence core in [MAND8 procurement readiness](mand8-procurement-readiness.md). They must not fork independent copies of the receipt, verifier, signer, or Lloyd's workflow evidence.

## 1. Shared Core Corpus
- MAND8 risk_receipt.v1 schema (frozen domain contract)
- ink.receipt.v2 schema (trust-waist contract)
- Trust registry and revocation list specifications
- Verification policy framework (bank_strict, host_compatibility)
- [ProofMail hosted workflow](proofmail-hosted-workflow.md)
- Lloyd's Labs demo scenario (lloyds_incident_to_renewal)

## 2. Investor Overlay
- Focus: Market opportunity and technical differentiation
- Key metrics: Verification accuracy, trust boundary integrity
- Evidence: Independent verification reports, security posture
- Narrative: How MAND8 makes AI risk insurable at scale
- Risk factors: Regulatory landscape, competitive positioning

## 3. Buyer Room Overlay
- Focus: Procurement requirements and integration path
- Key documents: Design partner brief, insurer/MGA overlay
- Evidence: Pilot request templates, verification instructions
- Narrative: How MAND8 integrates with existing underwriting workflows
- Risk factors: Implementation complexity, change management

## 4. Content Strategy
- Tier 1: Executive summary with key differentiators
- Tier 2: Technical architecture and trust model
- Tier 3: Implementation guides and pilot frameworks
- Tier 4: Detailed specifications and API references

## 5. Access Control
- Investor room: NDA-gated, time-limited access
- Buyer room: Procurement-gated, role-based permissions
- Shared corpus: Version-controlled, audit-logged access
- All artifacts: Cryptographically signed, integrity-verified

## 6. Update Cadence
- Weekly: Metric updates and new verification reports
- Monthly: Overlay content refresh and new pilot templates
- Quarterly: Full corpus review and schema version alignment
- Annual: Strategic narrative update and market positioning
